# 提示词工程完全指南

## 什么是提示词工程？

提示词工程是一门实证科学，涉及迭代和测试提示词以优化AI表现。掌握提示词工程，能让你与AI的协作效率提升数倍。

> **黄金法则**：把你的提示词给一位对该任务背景了解甚少的同事看，请他照做。如果他感到困惑，AI也会困惑。

---

## 第一部分：基础技巧

### 1. 清晰直接（Be Clear and Direct）

AI对清晰、明确的指令响应良好。明确指定期望的输出能显著提升结果质量。

**核心比喻**：把AI想象成一位才华横溢但新入职、缺乏工作流程和规范的同事。你越精确地解释需求，结果越好。

**最佳实践：**
- 明确指定期望的输出格式和约束条件
- 当步骤顺序或完整性很重要时，使用编号列表或项目符号提供顺序指令

**示例：**

```
❌ 模糊指令：
帮我写一篇关于AI的文章

✅ 清晰指令：
请写一篇800字的科普文章，主题是"大语言模型的工作原理"，目标读者是高中生。
文章需要包含：1）基本概念解释 2）至少两个生活化的类比 3）当前应用的3个实例
语言风格：轻松活泼，避免专业术语
```

### 2. 添加上下文（Add Context）

向AI解释**为什么**某行为很重要，帮助它更好地理解你的目标，提供更有针对性的响应。

**示例：**

```
❌ 缺少上下文：
请将回复格式化为表格

✅ 添加上下文：
请将回复格式化为表格，因为我需要将结果粘贴到电子表格中进行进一步分析。
这种表格格式能让我直接复制粘贴而无需二次处理。
```

### 3. 给AI设定角色（Give AI a Role）

在系统提示中设定角色，可让AI的行为和语气专注于你的用例。即使只有一句话也有效果。

**示例：**

```python
message = client.messages.create(
    model="your-model",
    max_tokens=1024,
    system="你是一位专注于Python的编程助手，擅长写简洁高效的代码，习惯用类型注解。",
    messages=[
        {"role": "user", "content": "如何按键值对字典列表排序？"}
    ],
)
```

### 4. 使用XML标签结构化提示词

XML标签帮助AI无歧义地解析复杂提示，特别是在指令、上下文、示例和变量输入混合时。

**最佳实践：**
- 使用一致、描述性的标签名
- 内容具有自然层次时嵌套标签

```xml
<instructions>
  请分析以下两篇文档的异同点
</instructions>

<documents>
  <document index="1">
    <source>技术白皮书A</source>
    <content>文档A的完整内容...</content>
  </document>
  <document index="2">
    <source>技术白皮书B</source>
    <content>文档B的完整内容...</content>
  </document>
</documents>

<output_format>
  请用表格形式呈现异同对比
</output_format>
```

## 第二部分：中级技巧

### 5. 有效使用示例（Use Examples Effectively）

示例是引导AI输出格式、语气和结构最可靠的方式之一。少量精心设计的示例（few-shot）能提升准确性和一致性。

**示例设计原则：**
- **相关性**：紧密贴合实际用例
- **多样性**：覆盖边缘情况，避免AI误学到非预期模式
- **结构化**：用`<example>`标签包裹示例，以便AI区分示例与指令

**推荐数量**：3-5个示例效果最佳

```xml
<examples>
  <example>
    <input>这家餐厅的服务太差了</input>
    <output>负面 | 服务质量 | 2/5</output>
  </example>
  <example>
    <input>产品功能很强大，但价格偏高</input>
    <output>中性 | 产品价值 | 3/5</output>
  </example>
  <example>
    <input>体验超出预期，强烈推荐！</input>
    <output>正面 | 整体体验 | 5/5</output>
  </example>
</examples>

现在请分析：<input>包装精美，物流很快，但商品有轻微瑕疵</input>
```

### 6. 分离数据与指令（Separating Data from Instructions）

当提示词中混合了指令和数据时，将它们清晰分离能显著提升AI的理解和执行能力。

```xml
<instructions>
  请对以下客户反馈进行情感分类和主题提取
</instructions>

<data>
  客户反馈内容...
</data>

<rules>
  - 情感分为：正面、负面、中性
  - 主题从预设列表中选择
  - 如果反馈涉及多个主题，全部列出
</rules>
```

### 7. 格式化输出与为AI预留空间（Formatting Output & Speaking for AI）

你可以控制AI输出的格式，也可以在提示词中预先写好AI回复的开头部分，引导其按照期望格式继续。

**四种控制格式的方法：**

1. **告诉AI做什么，而非不做什么**
   - 不好的做法："不要使用markdown"
   - 好的做法："请用流畅的散文段落撰写回复"

2. **使用XML格式指示器**
   ```
   请在<analysis>标签中输出分析，在<summary>标签中输出总结
   ```

3. **匹配提示与期望输出的风格**
   - 提示词的格式风格会影响AI的响应风格

4. **使用详细提示指定格式偏好**

### 8. 预认知：逐步思考（Precognition / Thinking Step by Step）

对于需要推理的复杂任务，要求AI先思考再作答，能显著提升准确率。

```
请逐步分析以下问题：
1. 首先，识别问题中的关键变量
2. 然后，建立变量之间的关系
3. 接着，逐步推导计算
4. 最后，给出结论并验证

在给出最终答案之前，请在<thinking>标签中展示你的推理过程。
```

## 第三部分：高级技巧

### 9. 长上下文提示（Long Context Prompting）

处理大型文档或数据密集型输入（20k+ tokens）时，需精心构建提示词：

**关键策略：**

| 策略 | 说明 | 效果 |
|------|------|------|
| 长文档放顶部 | 将长文档和输入放在提示词顶部，查询和指令放末尾 | 质量提升高达30% |
| XML标签结构化 | 多文档时用`<document>`标签包裹每篇文档 | 减少信息混淆 |
| 基于引用回答 | 要求AI先引用相关部分再执行任务 | 提高准确性 |

```xml
<documents>
  <document index="1">
    <title>年度报告</title>
    <content>长文档内容...</content>
  </document>
  <document index="2">
    <title>季度数据</title>
    <content>长文档内容...</content>
  </document>
</documents>

基于以上文档，请回答：两个报告中对2025年增长预测的差异是什么？
请先引用相关段落，再进行分析。
```

### 10. 避免幻觉（Avoiding Hallucinations）

AI有时会编造不存在的信息。以下方法可有效减少幻觉：

- **要求AI先引用来源再回答**
- **让AI承认不确定性**："如果不确定，请说明"
- **使用基于引用的回答模式**
- **提供参考资料并限制AI只能基于资料回答**

```xml
<instructions>
  请仅基于以下提供的信息回答问题。如果提供的信息不足以回答问题，
  请明确说明"根据提供的信息无法确定"，不要推测或编造。
  回答时请引用支持你结论的具体段落。
</instructions>

<reference_material>
  参考内容...
</reference_material>

<question>
  你的问题...
</question>
```

### 11. 构建复杂提示词（Building Complex Prompts）

对于行业级复杂场景，需要组合多种技巧：

**对话机器人示例：**

```xml
<role>
  你是一位专业的客户服务代表，负责处理电商平台的产品咨询和售后问题。
</role>

<tone>
  友好、专业、有耐心。遇到抱怨时表示理解，遇到不合理要求时礼貌拒绝。
</tone>

<knowledge_base>
  {{product_info}}
  {{return_policy}}
  {{shipping_info}}
</knowledge_base>

<rules>
  1. 回答前先确认理解了客户的问题
  2. 仅基于知识库信息回答，不确定时说明并建议人工服务
  3. 涉及退款/换货时，先确认订单信息
  4. 如果客户情绪激动，先表示理解再提供解决方案
</rules>

<examples>
  <example>
    <customer>我收到的商品和描述不一样</customer>
    <response>非常抱歉给您带来不便。我理解收到与预期不符的商品一定很令人失望。请问您方便提供订单号吗？我会帮您查看具体情况并安排退换。</response>
  </example>
</examples>
```

### 12. 提示词链（Prompt Chaining）

将复杂任务分解为多个步骤，每个步骤由一个专门的提示词处理：

```
步骤1：信息提取提示词 → 提取关键信息
步骤2：分析提示词 → 基于提取的信息进行分析
步骤3：总结提示词 → 生成最终报告

优势：
- 每步更专注，准确率更高
- 可在中间步骤检查质量
- 更容易调试和优化
```

## 第四部分：评估与优化

### 提示词评估（Prompt Evaluation）

在生产环境中，需要系统化地评估提示词质量：

1. **定义成功标准**：明确什么算"好的输出"
2. **构建测试集**：准备代表性的输入-期望输出对
3. **自动化评估**：使用评估框架批量测试
4. **迭代优化**：基于评估结果调整提示词

### 工具使用（Tool Use）

AI可以调用外部工具来扩展能力，关键实现要点：

- **工具定义清晰**：名称、描述、参数Schema完整
- **明确指令触发**：告诉AI何时使用哪个工具
- **并行调用优化**：独立的工具调用可以并行执行
- **结果处理**：指导AI如何理解和使用工具返回的结果
