# 提示词工程交互式教程精要

本教程提炼自官方交互式课程的9个章节，从基础到高级循序渐进。每章包含核心知识点和实战技巧。

---

## 第一章：基础提示词结构

### 核心概念

一个好的提示词包含两个核心部分：

1. **指令（Instruction）**：你想要AI做什么
2. **上下文（Context）**：帮助AI理解任务的背景信息

### 提示词结构模板

```
[角色设定] 你是一位专业的{领域}专家
[背景信息] 我们正在{具体场景}
[任务指令] 请{具体动作}
[输出格式] 请按以下格式输出：{格式说明}
[约束条件] 请注意：{限制和要求}
```

### 常见错误

| 错误类型 | 示例 | 修正 |
|----------|------|------|
| 指令模糊 | "帮我写点什么" | 明确具体写什么、给谁看、什么风格 |
| 缺少上下文 | "分析这个数据" | 说明数据来源、分析目的、期望结论 |
| 无格式要求 | "列出要点" | 指定用表格、列表还是段落 |

## 第二章：清晰与直接

### 原则：明确比含蓄好

AI不会"猜"你的意思。越直接、越明确，结果越好。

**改进前后对比：**

```
改进前：能不能帮我看看这个？
改进后：请检查以下代码中的bug，重点关注：
1. 空指针异常
2. 数组越界
3. 资源泄漏
```

### 编号列表的力量

当步骤顺序重要时，使用编号列表：

```
请按以下步骤处理：
1. 提取文本中的所有人名
2. 为每个人名标注出现的次数
3. 按出现次数从多到少排序
4. 输出前10个人名及其出现次数
```

## 第三章：角色设定

### 为什么角色设定有效？

角色设定帮助AI：
- 聚焦特定领域的知识
- 采用合适的语气和专业度
- 遵循特定领域的惯例和规范

### 角色设定技巧

**简短但精准：**
```
你是一位资深Python开发者，偏好函数式编程风格，注重代码可读性和性能。
```

**设定语气：**
```
你是一位耐心友好的编程老师，擅长用简单的类比解释复杂概念。
回答时用日常生活中的例子，避免过多专业术语。
```

**设定约束：**
```
你是一位安全审计专家。在审查代码时，必须：
- 标注每个安全风险的严重等级（高/中/低）
- 提供具体的修复建议
- 引用相关的安全标准和最佳实践
```

## 第四章：分离数据与指令

### 为什么要分离？

当提示词中混合了指令和数据时，AI可能混淆哪些是指令、哪些是要处理的数据。分离后：
- AI理解更准确
- 更容易复用模板
- 更换数据无需重写指令

### 使用XML标签分离

```xml
<instructions>
  请对以下客户反馈进行分类和情感分析
</instructions>

<feedback>
  客户反馈的原始内容放在这里...
</feedback>

<categories>
  产品质量、服务态度、物流配送、价格、其他
</categories>

<output_format>
  分类：___
  情感：正面/负面/中性
  关键词：___、___、___
</output_format>
```

### 变量化模板

```xml
<instructions>
  请将以下{{source_language}}文本翻译为{{target_language}}。
  保持原文的语气和风格，专业术语请保留原文并附上翻译。
</instructions>

<source_text>
  {{text_to_translate}}
</source_text>
```

## 第五章：格式化输出与为AI预留空间

### 控制输出格式

**方法1：明确指定格式**
```
请以JSON格式输出，包含以下字段：
{
  "summary": "50字以内的总结",
  "key_points": ["要点1", "要点2", "要点3"],
  "sentiment": "positive/negative/neutral",
  "confidence": 0.0到1.0之间的数值
}
```

**方法2：使用XML标签标记输出区域**
```
请在<analysis>中输出详细分析，在<conclusion>中输出最终结论。
```

**方法3：提供输出模板**
```
请按照以下模板输出：

## 摘要
[一段话总结]

## 关键发现
1. [发现1]
2. [发现2]

## 建议
- [建议1]
- [建议2]
```

### 为AI预留空间

在对话中预先写好AI回复的开头，引导其按期望格式继续：

```python
messages = [
    {"role": "user", "content": "列出5种编程语言的名称和特点"},
    {"role": "assistant", "content": "以下是5种编程语言的名称和特点：\n\n| 语言 |"}
    # AI会自然地继续填充表格
]
```

## 第六章：预认知（逐步思考）

### Chain-of-Thought的威力

让AI先展示推理过程，再做结论，能显著提升准确率。

**触发方式：**

1. **直接要求**
   ```
   请一步步思考并回答
   ```

2. **要求展示推理**
   ```
   在给出答案之前，请在<thinking>标签中展示你的推理过程
   ```

3. **提供推理示例**
   ```
   问题：一个商店有23个苹果，卖了15个，又进货了8个，现在有多少个？
   推理：初始有23个，卖出15个后剩23-15=8个，又进货8个后共8+8=16个
   答案：16个
   
   现在请用同样的方式回答：...
   ```

### 适合逐步思考的场景

| 场景 | 示例 |
|------|------|
| 数学计算 | 应用题求解 |
| 逻辑推理 | 谁在说谎问题 |
| 多步分析 | SWOT分析 |
| 复杂决策 | 方案评估与选择 |

## 第七章：使用示例

### Few-Shot Prompting

通过提供示例来引导AI的输出格式和风格。

**Zero-shot（无示例）：**
```
请对以下评论进行情感分类：这个产品太棒了！
```

**One-shot（一个示例）：**
```
请对评论进行情感分类：
示例：服务态度很差 → 负面

这个产品太棒了！
```

**Few-shot（多个示例）：**
```
请对评论进行情感分类和主题提取：

示例1：这个产品太棒了！ → 正面 | 产品质量
示例2：等了三天才到货 → 负面 | 物流配送
示例3：性价比一般，不功不过 → 中性 | 价格价值

输入：客服回复很及时，但解决问题的能力有待提升
```

### 示例设计原则

1. **相关**：示例应该贴近实际用例
2. **多样**：覆盖不同类型和边缘情况
3. **一致**：所有示例格式统一
4. **适量**：3-5个最佳，太少不够，太多浪费token

## 第八章：避免幻觉

### 幻觉的类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 事实性幻觉 | 编造不存在的事实 | 虚构论文引用 |
| 数字幻觉 | 篡改或编造数字 | 错误的统计数据 |
| 逻辑幻觉 | 推理过程有误 | 错误的因果推导 |

### 减少幻觉的策略

**策略1：限制信息来源**
```xml
<instructions>
  仅基于以下提供的资料回答问题。如果资料中没有相关信息，
  请明确回答"根据提供的信息无法确定"。
</instructions>

<reference>
  参考资料内容...
</reference>
```

**策略2：要求引用来源**
```
请在回答时引用支持你结论的具体段落或数据点。
如果无法引用，请说明这是你的推测。
```

**策略3：让AI承认不确定**
```
如果你的置信度低于80%，请在回答中标注[不确定]。
宁愿说"我不确定"，也不要编造答案。
```

**策略4：分步验证**
```
请按以下步骤回答：
1. 首先，列出你知道的相关事实
2. 然后，标注每个事实的来源和置信度
3. 最后，基于高置信度的事实给出结论
4. 如果关键信息缺失，明确列出缺失项
```

## 第九章：构建复杂提示词

### 行业场景：客服对话机器人

```xml
<role>
  你是一位专业的电商客服，负责处理产品咨询和售后问题。
</role>

<knowledge_base>
  {{product_info}}
  {{return_policy}}
  {{faq}}
</knowledge_base>

<rules>
  1. 回答前确认理解了客户问题
  2. 仅基于知识库信息回答
  3. 涉及退款/换货时先确认订单
  4. 情绪激动的客户先安抚
  5. 无法解决的问题转人工
</rules>

<tone>
  友好、专业、有耐心
</tone>

<examples>
  <example>
    <input>我收到的商品有损坏</input>
    <output>非常抱歉给您带来不便。请问您方便提供订单号和损坏照片吗？
    我会立即为您安排退换货处理。</output>
  </example>
  <example>
    <input>你们这个价格能不能再便宜点</input>
    <output>感谢您的关注！目前的价格已经是我们的最优价格了。
    不过我可以帮您查看是否有可用的优惠券，请问您是我们的会员吗？</output>
  </example>
</examples>
```

### 行业场景：法律文书分析

```xml
<role>
  你是一位法律研究助手，擅长分析合同条款和法律文书。
</role>

<task>
  请分析以下合同中的风险条款
</task>

<document>
  {{contract_text}}
</document>

<analysis_framework>
  请从以下维度分析每个条款：
  1. 风险等级：高/中/低
  2. 风险类型：责任风险/合规风险/商业风险
  3. 建议修改方案
  4. 参考法律依据
</analysis_framework>

<output_format>
  | 条款编号 | 内容摘要 | 风险等级 | 风险类型 | 修改建议 |
</output_format>
```

## 附录：超越基础提示

### 链式提示（Prompt Chaining）

将复杂任务拆解为多个步骤，每步由专门的提示词处理：

```
步骤1：提取 → 从原始文本中提取关键信息
步骤2：分类 → 对提取的信息进行分类
步骤3：总结 → 基于分类结果生成总结
```

### 工具使用

让AI调用外部工具来增强能力：

```python
tools = [
    {"name": "calculator", "description": "数学计算器"},
    {"name": "search", "description": "网络搜索"},
    {"name": "database", "description": "数据库查询"},
]
```

### 检索增强

结合外部知识库提供��确的上下文信息：

```
1. 用户提问
2. 从知识库检索相关文档
3. 将文档作为上下文加入提示词
4. AI基于检索到的内容回答
```
