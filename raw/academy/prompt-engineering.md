# Prompt Engineering 完全指南

**课程概述**

Prompt Engineering（提示词工程）是使用和优化AI的关键技能。本课程将系统教你如何编写高效Prompt。

## 第一章：什么是Prompt Engineering?

**Prompt Engineering** 是设计和优化提示词的艺术和科学，目标是：
- 获得更准确、更相关的回复
- 降低token消耗
- 提升用户体验

### 为什么重要？

**差的Prompt**：
```
写一个Python函数
```

**好的Prompt**：
```
写一个Python函数，实现快速排序算法。
要求：
1. 使用递归
2. 添加详细注释
3. 包含时间复杂度分析
4. 提供使用示例
```

## 第二章：Prompt结构

### 标准Prompt模板

```
{角色}：你是一个{专业角色}

{上下文}：我要完成{任务}，背景是{背景信息}

{指令}：请{具体行动}

{约束}：要求：
- {约束1}
- {约束2}

{示例}：例如：{示例输入} → {示例输出}

{输出格式}：请以{格式}输出
```

### 实际应用

**完整示例**：
```
你是一个Python编程专家，擅长算法和数据结构。

我要实现一个排序算法教学模块，背景是计算机科学入门课程。

请写一个快速排序的Python实现，要求：
1. 使用递归实现
2. 添加逐行注释
3. 分析时间复杂度和空间复杂度
4. 提供3个使用示例

例如：输入 [3,1,4,1,5] → 输出 [1,1,3,4,5]

请以Markdown格式输出，包含代码块和公式。
```

## 第三章：核心原则

### 原则1：具体明确 🎯

**差**：
```
帮我写点东西
```

**好**：
```
帮我写一篇500字的产品介绍，产品是智能手表，
目标用户是25-35岁的运动爱好者，
重点突出心率监测和GPS功能。
```

### 原则2：提供上下文 📚

**差**：
```
怎么优化性能？
```

**好**：
```
我有一个Python Web应用，使用Django框架，
数据库是PostgreSQL，目前有10万用户，
首页加载需要3秒，怎么优化性能？
请给出具体的代码建议。
```

### 原则3：使用示例 📝

**差**：
```
分类这些评论
```

**好**：
```
我要对用户评论进行情感分类。

示例：
输入："这个产品太棒了！" → 输出：正面
输入："质量很差，浪费钱" → 输出：负面
输入："还行吧，不太好也不太差" → 输出：中性

现在请分类："客服态度很好，但物流太慢"

请以JSON格式输出：{"sentiment": "正面/负面/中性", "confidence": 0.95}
```

### 原则4：分步思考 🪜

**差**：
```
分析这个代码
```

**好**：
```
请逐步分析以下代码：

步骤1：理解代码功能（用一句话描述）
步骤2：识别主要逻辑（画出流程图）
步骤3：找出潜在问题（bug、性能、安全）
步骤4：提出改进建议（给出具体代码）

代码：
```python
{def buggy_function(x):
    return 1/x}
```
```

## 第四章：高级技巧

### 技巧1：Few-Shot Learning（少样本学习）

**Zero-shot**（无示例）：
```
将以下文本翻译成法语：Hello, how are you?
```

**Few-shot**（有示例）：
```
将以下文本翻译成法语：

示例1：Hello → Bonjour
示例2：Thank you → Merci
示例3：Goodbye → Au revoir

现在翻译：How are you?
```

**经验**：2-5个示例最佳

### 技巧2：Chain-of-Thought（思维链）

**直接回答**：
```
问题：Roger有5个网球，他又买了2罐，每罐3个。他现在有几个网球？

回答：11个。
```

**思维链**：
```
问题：Roger有5个网球，他又买了2罐，每罐3个。他现在有几个网球？

让我们一步步思考：
1. Roger最初有5个网球
2. 他买了2罐
3. 每罐有3个，所以2罐共有 2 × 3 = 6个
4. 总共：5 + 6 = 11个

答案：Roger现在有11个网球。
```

**效果**：复杂推理任务准确率提升30-50%

### 技巧3：角色扮演

**无角色**：
```
解释什么是区块链
```

**有角色**：
```
你是一个区块链专家，擅长向非技术人员解释复杂概念。
请用类比和简单语言解释什么是区块链，
假设你在跟一个完全不懂技术的朋友解释。
```

### 技巧4：输出格式控制

**无格式控制**：
```
列出Python常用库
```

**有格式控制**：
```
列出Python数据科学常用库，请以以下格式输出：

| 库名 | 用途 | 安装命令 |
|------|------|----------|
| pandas | 数据分析 | pip install pandas |
| numpy | 数值计算 | ... |
```

## 第六章：调试和优化

### 问题1：回复不相关

**原因**：
- Prompt太模糊
- 缺少上下文

**解决**：
```python
# 差
prompt = "解释API"

# 好
prompt = """
你是一个技术文档作者。

我要为新手开发者解释什么是REST API。

要求：
1. 用类比解释（比作餐厅点餐）
2. 包含HTTP方法说明
3. 提供一个简单示例（使用curl命令）
4. 字数：300-500字

输出格式：Markdown
"""
```

### 问题2：回复太冗长

**解决**：
```
{原Prompt}

重要：请将回答控制在200字以内，直接进入重点。
```

或：
```
{原Prompt}

输出格式：只列出要点，不要详细解释。
```

### 问题3：幻觉（编造信息）

**解决**：
```
{原Prompt}

注意：如果你不确定某个信息，请明确说明"不确定"或"需要查证"，
不要编造。只使用你确定正确的信息。
```

### 问题4：格式不对

**解决**：
```
{原Prompt}

输出格式（严格遵循）：
{
  "summary": "简短摘要",
  "key_points": ["要点1", "要点2"],
  "action_items": ["行动项1", "行动项2"]
}

必须输出合法JSON，不要包含markdown代码块标记。
```

## 第八章：评估Prompt质量

### 评估维度

1. **准确性**：输出是否正确答案
2. **一致性**：相似输入是否产生相似输出
3. **鲁棒性**：微小变化是否影响输出
4. **效率**：是否使用最少token
5. **可用性**：输出是否易于使用

### A/B测试

**版本A**：
```
总结这篇文章
```

**版本B**：
```
请总结以下文章：

要求：
1. 长度：100-150字
2. 包含3个要点
3. 客观中立，不添加个人观点

文章：{文章}
```

**评估**：运行10次，比较输出质量

## 第九章：Prompt版本管理

### 最佳实践

1. **版本化**：`prompt_v1.0.md`, `prompt_v1.1.md`
2. **记录变更**：每次修改记录原因和效果
3. **回滚机制**：保留旧版本，必要时回滚
4. **文档化**：记录每个Prompt的适用场景

## 第十章：安全和伦理

### 安全风险

1. **Prompt注入**：用户绕过系统Prompt
   - 防御：输入验证、输出过滤

2. **敏感信息泄露**：Prompt包含密钥/密码
   - 防御：永远不要将密钥写在Prompt中

3. **越权访问**：用户通过Prompt获取未授权信息
   - 防御：实现权限检查

### 伦理考虑

1. **偏见**：Prompt可能导致偏见输出
   - 解决：使用中性语言

2. **误导**：故意设计Prompt产生误导信息
   - 解决：遵循伦理准则

3. **透明度**：用户应知道在与AI交互
   - 解决：明确标识AI生成内容

## 总结

**Prompt Engineering核心要点**：

1. ✅ **具体明确**：细节决定质量
2. ✅ **提供上下文**：背景信息很重要
3. ✅ **使用示例**：Few-shot效果显著
4. ✅ **分步思考**：Chain-of-Thought提升推理
5. ✅ **迭代优化**：Prompt是写出来的，不是一次成型的
