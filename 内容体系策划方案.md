## 一、内容分类体系

将全部资源按知识领域和学习层次划分为 **6大分类**，每个分类下设子类，确保资源归属清晰、无交叉歧义。

### 1.1 分类总览

| 分类编号 | 分类名称 | 定位 | 资源数量 |
|---------|---------|------|---------|
| C1 | 入门基础 | AI概念认知与基础操作 | 4门 |
| C2 | 提示词工程 | 提示词设计、评估与优化 | 7门+文档 |
| C3 | API与开发实战 | API调用、云部署、MCP协议 | 5门 |
| C4 | 智能体系统 | 智能体技能、子智能体、系统架构 | 3门 |
| C5 | AI素养与伦理 | 行业场景AI素养与伦理安全 | 7门 |
| C6 | 视频精选 | 深度讨论、技术解析、实战演示 | 170个视频（精选重组） |

## 二、学习路径设计

设计 **5条学习路径**，覆盖不同起点和目标人群，每条路径标注预计学习时长和核心产出。

### 路径总览

| 路径编号 | 路径名称 | 目标人群 | 预计时长 | 核心产出 |
|---------|---------|---------|---------|---------|
| P1 | 零基础AI入门 | AI小白、职场新人 | 2-3周 | 能独立使用AI助手完成日常工作 |
| P2 | 提示词工程精通 | 内容创作者、产品经理、运营 | 3-4周 | 掌握系统化提示词设计与评估方法 |
| P3 | AI应用开发者 | 程序员、技术创业者 | 4-6周 | 能独立构建和部署AI应用 |
| P4 | 智能体系统架构师 | 高级开发者、AI工程师 | 3-4周 | 能设计和实现多智能体协作系统 |
| P5 | AI素养推广者 | 教师、培训师、管理者 | 2-3周 | 能在组织内推广AI素养教育 |

  └─ C6 视频精选·AI原理图解（选看3-5集）

第2周：动手实践
  ├─ C1.2 AI助手日常使用 ── 学会日常对话式使用
  ├─ C1.3 AI协作与文件处理 ── 学会用AI处理文档和数据
  └─ C2.1 提示词基础（前3章）── 初级提示技巧

第3周：效率提升
  ├─ C2.2 通用提示原则 ────── 掌握6大提示原则
  ├─ C2.3 输出格式化 ──────── 控制AI输出质量

## 三、内容改写规范（去品牌化原则）

### 3.1 核心原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **知识纯净** | 所有内容只传递知识，不引导至任何品牌产品 | ❌ "使用Claude的Artifacts功能" → ✅ "使用AI的结构化输出功能" |
| **术语通用化** | 将品牌专属术语替换为行业通用术语 | ❌ "Claude API" → ✅ "大模型API"；❌ "Claude Sonnet" → ✅ "中量级模型" |
| **场景普适化** | 操作示例脱离特定产品界面，改为通用流程描述 | ❌ "在Claude.ai对话框中" → ✅ "在AI助手的对话界面中" |
| **方法论优先** | 强调"为什么这样做"而非"用哪个产品做" | 每个知识点先讲原理和方法论，再给通用操作步骤 |
| **多方案呈现** | 涉及工具选择时，提供2-3种通用方案 | 不只推荐一个品牌，提供开源/商业多选项 |

### 3.2 具体改写规则

#### 术语替换表

| 原始术语 | 替换为 |
|---------|--------|
| [品牌名] API | 大模型API / LLM API |
| [品牌名] 助手 | AI助手 |
| [品牌名] 编程助手 | AI编程助手 |
| [品牌名] 平台 | AI应用平台 |
| [品牌名] Sonnet/Opus/Haiku | 中量级模型/重量级模型/轻量级模型 |
| [品牌名] Console | API控制台 |
| [品牌名] Artifacts | 结构化输出 / 代码沙盒 |
| [品牌名] Projects | 项目工作区 |
| extended thinking | 深度推理模式 |
| tool_use | 工具调用 / 函数调用 |

#### 内容元素处理

| 元素 | 处理方式 |
|------|---------|
| 产品截图 | 替换为通用示意图或抽象流程图 |
| 品牌Logo/图标 | 全部移除，替换为通用图标 |
| 品牌色彩/设计语言 | 替换为站点自有设计系统 |
| 品牌案例 | 替换为虚构但真实的行业案例 |
| 品牌文档链接 | 替换为站内知识卡片或通用技术文档链接 |
| 定价/套餐信息 | 全部移除 |
| 竞品对比 | 移除对比表述，改为功能特性客观描述 |
| 品牌代言人/讲师 | 替换为"AI教程精华站编辑团队" |

### 3.3 改写质量检查清单

每篇内容上线前，须通过以下检查：

- [ ] 全文搜索品牌名，零命中
- [ ] 无品牌专属术语残留
- [ ] 截图/图片中无品牌元素
- [ ] 示例代码中无品牌专属SDK包名（改为通用HTTP或通用SDK）
- [ ] 外部链接不指向品牌商业页面
- [ ] 内容价值不因去品牌化而损失（知识完整性检查）
- [ ] 方法论描述准确，未因去品牌化而歪曲原意
- [ ] 多方案建议客观中立，无隐性偏向

按 **5个梯队** 安排上线节奏，确保首屏即有价值、持续有增量。

|--------|------|------|
| ★★★ | C1.1 AI能力与局限 | 零基础用户的第一站，认知基石 |
| ★★★ | C1.2 AI助手日常使用 | 最大潜在用户群的入口 |
| ★★★ | C2.1 提示词基础（交互式教程） | 核心差异化内容，9章完整体系 |
| ★★★ | C2.2 通用提示原则 | 最具传播力的干货，6大原则 |
| ★★★ | P1 零基础AI入门路径 | 降低入门门槛的核心路径 |
| ★★★ | 提示词模板库（知识卡片） | 高频搜索需求，SEO入口 |

### P1 · 核心内容（第3-4周）

> 构成站点内容深度的骨架
